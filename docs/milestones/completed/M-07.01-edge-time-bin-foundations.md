# M-07.01 - Edge Time Bin Foundations (Schema + Artifacts + Simulation)

**Status:** ✅ Complete  
**Dependencies:** ✅ M-06.02 (Model Compiler), ✅ CL-M-04.04 (Classes and Routing), ✅ SB-M-05.04 (ServiceWithBuffer)  
**Target:** Introduce edge time-bin metrics in schemas, run artifacts, and simulation outputs so edges become first-class flow facts.

---

## Overview

Edge time bins elevate edges from topology-only links to quantitative flow carriers. This milestone adds the foundational contract and engine emission for edge metrics in synthetic runs, enabling consistent edge data across artifacts and APIs without changing templates.

### Strategic Context
- **Motivation:** Node-only metrics cannot explain multi-parent routing, retries, or true downstream flow. Edge facts are required for truthful overlays and later path analytics.
- **Impact:** New `edges` section in time-travel state and new edge CSVs in run artifacts. Simulation runs emit per-edge flow totals (and per-class when available).
- **Dependencies:** Builds on the evaluation integrity compile-to-DAG pipeline, and on class-aware flows and service-with-buffer semantics.

## Scope

### In Scope ✅
1. Extend time-travel schemas to include edge metrics in `/state` and `/state_window` responses.
2. Extend run artifact series layout to include edge series and index metadata.
3. Emit EdgeTimeBins for simulation runs (flow totals and per-class breakdowns when class data is present).
4. Wire API endpoints to surface edges when present (no UI changes yet).

### Out of Scope ❌
- UI overlays or edge visualization changes.
- Telemetry ingestion for edges (handled in the Telemetry Ingestion epic).
- Conservation analyzers or quality flags (handled in M-07.02).
- Path-level analytics or process-mining views.

## Requirements

### Functional Requirements

#### FR1: Edge time-bin schema support
**Description:** Add an `edges` section alongside `nodes` in the time-travel state schema.

**Acceptance Criteria:**
- [x] `docs/schemas/time-travel-state.schema.json` includes `edges` with per-bin metrics.
- [x] Edge entries support `flowVolume` and `byClass` (optional) fields.
- [x] Existing node-only responses remain valid (edges are optional).
 

**Error Cases:**
- Missing edges must not break parsing; clients treat `edges` as optional.

#### FR2: Run artifact edge series
**Description:** Persist edge series in canonical run artifacts with stable indexing.

**Acceptance Criteria:**
- [x] `series/index.json` lists edge series (edge id + metric key + optional class).
- [x] Edge CSVs are emitted under `series/` using the canonical naming scheme.
- [x] Edge series align to the same bin grid as node series.

**Error Cases:**
- If a run does not emit edges, `index.json` omits edge entries without error.

#### FR3: Simulation emits edge flows
**Description:** Simulation runs accumulate per-edge flow totals and per-class flows when class data exists.

**Acceptance Criteria:**
- [x] Edge flow totals per bin match routed served volumes from upstream nodes.
- [x] For classed flows, edge `byClass` values are emitted for those classes.
- [x] Retry or error flows are not introduced in this milestone (flow totals only).

**Error Cases:**
- If class attribution is unavailable, emit edge totals without `byClass` instead of zeros.

#### FR4: API surfaces edge data
**Description:** `/state` and `/state_window` include edges when available.

**Acceptance Criteria:**
- [x] `/state` and `/state_window` responses include `edges` when present in artifacts.
- [x] Edge data respects window and bin selection.

### Non-Functional Requirements

#### NFR1: Backward compatibility
**Target:** Node-only clients continue to function without changes.
**Validation:** Existing UI and API tests pass with edges absent.

#### NFR2: Deterministic output
**Target:** Edge series are deterministic for simulation runs.
**Validation:** Golden fixtures stay stable across runs for the same seed/model.

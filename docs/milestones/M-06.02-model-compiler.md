# M-06.02 — Model Compiler (Compile-to-DAG Extraction)

**Status:** ✅ Complete  
**Dependencies:** ✅ M-06.01 (Evaluation Integrity contract)  
**Target:** Extract compile-to-DAG expansion into a core compiler so all entry points use the same pure evaluation pipeline.

---

## Overview

M-06.01 moved derived-series injection (queue depth, retry echo, class overrides) into a compile-to-DAG step, but that step currently lives in the artifact writer. This milestone extracts it into a **core compiler** so the generator, artifact writer, and analysis codepaths all share a single, explicit pipeline.

The user-facing model remains node-based. The compiler expands internal nodes (hidden series) into an internal DAG and returns a canonical, evaluation-ready model.

---

## Scope

### In Scope ✅
1. Create a `ModelCompiler` (or equivalent) in `FlowTime.Core` that:
   - Expands queue depth nodes for queue-like topology nodes.
   - Expands retry echo nodes using the retry kernel policy.
   - Preserves topology semantics and initial conditions.
2. Use the compiler in:
   - Artifact writer (run exports).
   - Generator evaluation pipeline.
   - Template invariant analysis (where applicable).
3. Ensure all compilation paths remain deterministic and DAG-safe.
4. Add tests that assert the compiled model produces identical results across entry points.

### Out of Scope ❌
- Template changes or schema changes.
- UI modifications.
- New node kinds or routing behaviors.

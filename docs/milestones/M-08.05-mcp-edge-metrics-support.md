# M-08.05 - MCP Edge Metrics Support

**Status:** Planned  
**Dependencies:** M-07.04 (Edge Semantics Contract), M-07.05 (Derived Path Latency)  
**Target:** Extend MCP analyst tooling to expose edge-level metrics and quality signals from the time-travel API.

---

## Overview

Edge metrics (edge time bins, edge quality, and edge warnings) are now part of the FlowTime contract under the edge-time-bin epic. The MCP server currently exposes only node-centric state. This milestone updates MCP schemas and tooling so AI clients can query and interpret edge series without re-deriving anything client-side.

### Strategic Context
- **Motivation:** MCP clients should consume the same engine-owned edge semantics as the UI.
- **Impact:** Enables edge-aware analysis, conservation checks, and retry-load interpretation in MCP workflows.
- **Dependencies:** Requires stable edge semantics and quality definitions from the edge-time-bin epic.

---

## Scope

### In Scope
1. MCP tool schema updates to include edge series in `get_state_window`.
2. Optional edge filters (`edgeIds`, `edgeMetrics`, `classIds`) aligned with API support.
3. Edge quality and edge warning metadata surfaced in MCP responses.
4. Documentation updates for MCP analyst workflow and tool schemas.

### Out of Scope
- New edge computations or inference in MCP (engine remains source of truth).
- UI changes or new UI endpoints.
- Path-level analytics beyond the edge-time-bin contract.

---

## Requirements

### Functional Requirements

#### FR1: Edge Series Exposure
**Description:** MCP `get_state_window` returns edge metrics when present in the run.

**Acceptance Criteria:**
- [ ] Edge series are included alongside node series when available.
- [ ] Edge series are keyed by `edgeId` and optionally `classId`.
- [ ] Tool response makes it explicit when edge series are missing or approximate.

#### FR2: Edge Quality + Warnings
**Description:** Edge quality and edge warning signals are included in MCP outputs.

**Acceptance Criteria:**
- [ ] Edge quality is returned for each edge when present in `/state_window`.
- [ ] Edge warnings are surfaced in MCP `get_run_summary` or `get_state_window` output.

#### FR3: Edge Filters
**Description:** MCP tools accept edge-specific filters to reduce payloads.

**Acceptance Criteria:**
- [ ] `edgeIds` filter is supported when the API provides it.
- [ ] `edgeMetrics` filter is supported when the API provides it.

### Non-Functional Requirements

#### NFR1: No Client-Side Derivation
**Target:** MCP server never infers edge metrics; it only forwards engine output.
**Validation:** Tool responses map 1:1 to API payloads.

**Goal:** Define MCP schema changes and update analyst workflow docs.

### Phase 2: MCP Server + API Wiring
**Goal:** Pass through edge filters and return edge series in tool responses.

**Tasks:**
1. Update MCP server `get_state_window` handler to accept edge filters.
2. Map API response edge sections into MCP tool output.
3. Add contract tests for edge-series presence when available.

**Success Criteria:**
- [ ] MCP `get_state_window` returns edge series for edge-enabled runs.

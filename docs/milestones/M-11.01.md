# M-11.01 — Net 9 Package Update Epic

**Status:** 📋 Planned  
**Dependencies:** None  
**Target:** Update non-UI .NET dependencies to the latest .NET 9-compatible releases (UI/MudBlazor deferred to M-11.02).

---

## Overview

This milestone updates FlowTime's non-UI .NET dependencies to the latest compatible .NET 9 releases in a controlled, phased sequence. It focuses on stability and compatibility, sequencing updates by core libraries, services/CLI, and tests. UI updates are explicitly deferred to M-11.02 to avoid MudBlazor regressions.

### Strategic Context
- **Motivation**: Reduce drift from current .NET 9 patch/minor releases and minimize security/compat risk.
- **Impact**: Cleaner upgrade path, fewer breaking changes, and improved build/test stability.
- **Dependencies**: None, but UI work must be last.

## Scope

### In Scope ✅
1. Update all non-UI .NET package references to the latest .NET 9-compatible versions.
2. Keep TargetFramework at `net9.0` for all projects.
3. Separate major-version package updates into dedicated sub-phases with review.
4. Defer UI updates (MudBlazor, WASM stack, bUnit) to M-11.02.

### Out of Scope ❌
- ❌ Upgrading to .NET 10 packages or changing TargetFramework.
- ❌ Functional changes unrelated to dependency updates.
- ❌ Any UI/MudBlazor updates (tracked in M-11.02).

### Future Work (Optional)
- M-11.02 for UI/MudBlazor package updates.
- Consider .NET 10 upgrade after UI stability pass.

## Requirements

### Functional Requirements

#### FR1: Net 9-compatible package updates

**Description:** All top-level package references are updated to the latest .NET 9-compatible versions, using `--highest-minor` where needed.

**Acceptance Criteria:**
- [ ] All Microsoft.* packages stay on `9.0.x`.
- [ ] No TargetFramework changes from `net9.0`.
- [ ] All package upgrades are documented per project.

**Examples:**
- Updating `Microsoft.Extensions.*` from `9.0.0` to the latest `9.0.x`.

**Error Cases:**
- Any `10.x` dependency is introduced.

#### FR2: UI updates are deferred

**Description:** UI package updates are excluded from this milestone and handled in M-11.02.

**Acceptance Criteria:**
- [ ] No UI packages are updated in M-11.01.
- [ ] M-11.02 is created to track UI updates.

### Non-Functional Requirements

#### NFR1: Build and test stability

**Target:** All relevant tests pass after each phase.
**Validation:** Run phase-specific tests; full suite at milestone end.

## Implementation Plan

### Phase 1: Audit and Baseline

**Goal:** Establish the upgrade plan and baseline package list.

**Tasks:**
1. Capture current and target package versions per project.
2. Confirm all updates remain on .NET 9-compatible versions.
3. Prepare tracking doc with test-first tasks.

### Phase 2: Core Libraries

**Goal:** Update core and contracts packages.

**Tasks:**
1. Update `FlowTime.Contracts`, `FlowTime.Core`, `FlowTime.Sim.Core`.
2. Run core/unit tests.

### Phase 3: Services and CLI

**Goal:** Update API and CLI dependencies.

**Tasks:**
1. Update `FlowTime.API`, `FlowTime.Cli`, `FlowTime.Sim.Service`, `FlowTime.Sim.Cli`.
2. Run API/CLI tests.

### Phase 4: Non-UI Tests

**Goal:** Update test project dependencies and validate.

**Tasks:**
1. Update test packages (xUnit, coverlet, test SDK, etc.).
2. Run non-UI test suites.

# M-11.02 — Net 9 UI Package Updates (MudBlazor)

**Status:** ✅ Complete  
**Dependencies:** ✅ M-11.01  
**Target:** Update UI packages to the latest .NET 9-compatible releases with MudBlazor upgraded last after all other UI dependencies are validated.

---

## Overview

This milestone updates the UI package stack (Blazor WASM, UI test dependencies, and MudBlazor) to the latest .NET 9-compatible versions. UI updates are isolated here to reduce risk, with MudBlazor upgraded last after the rest of the UI stack is validated.

### Strategic Context
- **Motivation**: UI packages often introduce behavioral changes; isolating them reduces risk.
- **Impact**: Up-to-date UI dependencies with validated rendering and tests.
- **Dependencies**: ✅ M-11.01 (non-UI updates completed).

## Scope

### In Scope ✅
1. Update non-MudBlazor UI package references in `FlowTime.UI` and `FlowTime.UI.Tests`.
2. Upgrade MudBlazor last, after other UI updates are validated.
3. Keep TargetFramework at `net9.0`.
4. Validate UI tests and smoke-check key UI flows.

### Out of Scope ❌
- ❌ Upgrading to .NET 10 packages or changing TargetFramework.
- ❌ UI redesigns unrelated to dependency updates.

### Future Work (Optional)
- Consider a .NET 10 UI uplift after validation.

## Requirements

### Functional Requirements

#### FR1: UI packages updated to latest net9-compatible

**Description:** Update MudBlazor, Blazor WebAssembly, and UI test packages to their latest .NET 9-compatible versions.

**Acceptance Criteria:**
- [ ] MudBlazor updated to latest net9-compatible version.
- [ ] Microsoft.AspNetCore.Components.WebAssembly packages updated to `9.0.x`.
- [ ] bUnit updated and UI tests pass.

**Error Cases:**
- Any `10.x` dependency is introduced.

### Non-Functional Requirements

#### NFR1: UI stability

**Target:** UI tests pass with no regressions.
**Validation:** Run UI tests and perform a brief UI smoke check.

## Implementation Plan

### Phase 1: UI Package Updates (Non-MudBlazor)

**Goal:** Update UI and UI test packages excluding MudBlazor.

**Tasks:**
1. Update `FlowTime.UI` and `FlowTime.UI.Tests` package references excluding MudBlazor.
2. Run UI tests and validate.

### Phase 2: MudBlazor Update (Last)

**Goal:** Upgrade MudBlazor after the rest of the UI stack is stable.

**Tasks:**
1. Update MudBlazor package reference.
2. Run UI tests and validate.

# M-07.06 Release Notes

**Release Date:** 2026-01-21  
**Milestone:** M-07.06 - Transit Node Modeling (Remove Edge Lag)  
**Status:** ✅ Complete  

---

## 🎯 Milestone Goal

Eliminate edge-level lag by modeling transit delays as explicit nodes.

## ✅ What's New

### Template Migration
- Replaced lagged edges with explicit transit nodes in transportation and incident-retry templates.
- Transit delays now appear as inspectable service nodes (no queues by default).

### Regression Coverage
- Added tests that fail if any first-party template still uses edge lag.
- Updated template docs to require transit nodes instead of edge behavior.

## 🧪 Test Coverage

- Per-project test suite (build + `dotnet test --nologo` per project).
- `EdgeLagTemplateTests`, `TransitNodeTemplateTests`

## 🔧 Implementation Highlights

- Removed `lag` from template edges; added transit nodes and edges to preserve timing.
- Added hidden zero processing-time series where needed to keep invariants clean.

## 📝 Migration Notes

- Edge lag is no longer used in first-party templates. Model transit delays as explicit nodes.

# Release M-06.01 — Evaluation Integrity (DAG/Spreadsheet Contract)

**Release Date:** 2026-01-18  
**Type:** Milestone delivery  
**Canonical Run:** TBD

## Overview

M-06.01 enforces a strict DAG/Spreadsheet evaluation contract in the engine. Overrides and derived series are now compiled into the evaluation pipeline so downstream series always recompute in order, with guardrails that flag late injection.

## Key Changes

1. **DAG-ordered class propagation + derived series**  
   Class overrides and derived queue/retry series now flow through the compiled DAG evaluation path instead of post-eval mutation.  
   (`src/FlowTime.Core/Artifacts/RunArtifactWriter.cs`, `src/FlowTime.Core/Artifacts/ClassContributionBuilder.cs`)
2. **Integrity guardrails + diagnostics**  
   Added invariant checks for late injection and class coverage gaps, with clearer warning behavior for zero-series placeholders.  
   (`src/FlowTime.Core/Analysis/InvariantAnalyzer.cs`)
3. **Schema + template alignment**  
   Series semantics now allow `origin: explicit`, and templates align node semantics for error series expectations.  
   (`docs/schemas/time-travel-state.schema.json`, `templates/*.yaml`)
4. **UI provenance clarity**  
   Inspector properties are grouped into Series vs Derived, with queue depth aliasing moved into tooltips.  
   (`src/FlowTime.UI/Pages/TimeTravel/Topology.razor`)

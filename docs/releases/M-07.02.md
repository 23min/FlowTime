**Status:** ✅ Complete

## 🎯 Milestone Goal

Add soft conservation checks for edge flows and surface explicit edge data quality in API metadata and run diagnostics.

## ✅ What's New

### Edge Conservation Diagnostics
- **Analyzer warnings:** Outgoing/incoming edge flow conservation checks with per-class coverage warnings.
- **Edge labels in warnings:** Diagnostics include edge identifiers for faster triage.
- **Run artifacts:** Conservation warnings flow into `run.json` diagnostics.

### Edge Quality Metadata
- **State metadata:** `edgeQuality` added to `/state` and `/state_window` metadata (`exact`, `approx`, `missing`, `partialClass`).
- **Schema/contracts:** Time-travel schema and contracts updated for edge quality.

### Test Stability
- **CLI integration test:** Avoids hanging `dotnet run` by executing the built CLI directly.
- **CI tests:** Per-project test steps with `--blame-hang` to prevent long full-suite timeouts.

# M-08.04 - MCP Storage Abstraction

**Status:** Complete  
**Release Date:** 2026-01-19  
**Branch:** `milestone/m-08.04`

## Overview

M-08.04 removes local filesystem dependencies from MCP modeling and analysis workflows. Drafts, generated models, and run bundles are now stored and referenced via storage-backed APIs, enabling HTTP-only deployments and multi-user readiness.

### Highlights
- **Pluggable storage layer:** Storage contracts and backends (filesystem, blob+index, blob+db) with storage references.
- **HTTP-first drafts:** Draft CRUD flows are now storage-backed and accessed via Sim endpoints.
- **Storage-backed models + runs:** Model generation returns `modelRef`, run orchestration returns `bundleRef`, and API run import accepts storage refs.
- **MCP server refactor:** Draft and run tools use HTTP storage endpoints, no repo paths required.

## API Changes

- **New Sim endpoints:**
  - `GET /api/v1/drafts`
  - `POST /api/v1/drafts`
  - `GET /api/v1/drafts/{draftId}`
  - `PUT /api/v1/drafts/{draftId}`
  - `DELETE /api/v1/drafts/{draftId}`
  - `GET /api/v1/templates/{id}/source`
- **Updated Sim responses:**
  - `POST /api/v1/drafts/generate` now returns `modelRef`
  - Run orchestration responses include `bundleRef`
- **Updated Engine API:**
  - `POST /v1/runs` accepts `bundleRef`

## Tooling Updates

- MCP draft tools now operate through Sim draft storage endpoints.
- MCP run tools import bundles via storage refs instead of local paths.

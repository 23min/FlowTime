# M-07.03 Release Notes

**Release Date:** 2026-01-19  
**Milestone:** M-07.03 - Edge Time Bin UI Overlays and UX  
**Status:** ✅ Complete  

---

## 🎯 Milestone Goal

Use edge time bins in the UI for overlays and tooltips, with clear fallback indicators when edge data is missing or approximate.

## ✅ What's New

### Edge Overlays & Tooltips
- Edge overlays use edge series when available, with per-class handling.
- Edge tooltips show flow, retries, total load, and quality notes when applicable.
- Fidelity indicator surfaces when edge quality is approximate or missing.

### Retry + Warning UX
- Retry badge appears on edges when retry load is present; node retry badge is suppressed by default.
- Edge warning badges highlight only edges that have edge-level warnings.

### Scrubber Clarity
- Bin number is shown with the timestamp for easier correlation when validating edge flows.

## 🧪 Test Coverage

- `dotnet test --nologo tests/FlowTime.UI.Tests/FlowTime.UI.Tests.csproj`

## 🔧 Implementation Highlights

### UI
- Edge overlay pipeline now prefers edge metrics over node-derived fallbacks.
- Edge warning styling is aligned to warning chip colors when edge warnings exist.

### Core/API
- Edge lag now emits a warning to discourage edge-level behavior; transit should be modeled as nodes.
- Edge quality treats router class subsets as exact when edge class coverage is satisfied.

## 📝 Migration Notes

- No schema changes in this milestone.
- Edge semantics contract and explicit edge metrics are planned in M-07.04.

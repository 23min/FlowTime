# M-10.02 - Dependency Constraints (Attached to Services)

**Status:** Complete  
**Release Date:** 2026-01-23  
**Branch:** `milestone/m-10.02`

## Overview

M-10.02 delivers Option B dependency constraints: shared constraint resources attached directly to service nodes. The engine now allocates constraint capacity proportionally per bin, exposes constraint series and status in state responses, and the UI surfaces constraint chips and charts for inspection.

### Highlights
- **Constraint registry + attachments:** services reference shared constraints without adding graph nodes.
- **Proportional allocation:** constraint capacity is allocated by per-bin demand across attached nodes.
- **Constraint series & status:** `/state` and `/state_window` include constraint arrivals/served/errors plus derived shortfall and limit flags.
- **UI inspection:** constraint chips (e.g., `C | db_main`) and constraint charts are visible in the inspector.

## Contract Updates

- **Constraint series in state/state_window**
  - `constraints.{id}.series`: `arrivals`, `served`, optional `errors`, optional `latencyMinutes`, derived `shortfall`
  - `constraintStatus.{id}`: per-bin flag indicating the node is limited by the constraint
- **Series metadata**
  - Explicit series tagged `origin: explicit`
  - Derived `shortfall` tagged `origin: derived`

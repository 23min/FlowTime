# M-10.01 - Dependency Constraints Foundations

**Status:** Complete  
**Release Date:** 2026-01-22  
**Branch:** `milestone/m-10.01`

## Overview

M-10.01 introduces the dependency node contract (Option A) so downstream/shared resources can be modeled explicitly in the DAG. This provides arrivals/served dependency signals in API outputs with warnings for missing inputs and a minimal reference template.

### Highlights
- **Dependency node kind:** `dependency` nodes now appear in `/state` and `/state_window` with arrivals/served/errors.
- **Edge pressure signals:** effort edges emit `flowVolume` and derived `retryVolume` to show actual flow + retry pressure.
- **Flow latency propagation:** flow latency now propagates across effort/dependency edges when flow volume exists.
- **Warnings:** new info warnings surface missing dependency arrivals/served signals and missing dependency effort/attempt coverage.
- **Reference template:** added a minimal dependency constraints template for validation.

## Contract Updates

- **Dependency node semantics**
  - Required series: `arrivals`, `served`
  - Optional: `errors`
  - No queue/capacity/retry semantics for dependency nodes in the MVP
- **Dependency enforcement warnings (info)**
  - `missing_dependency_arrivals`
  - `missing_dependency_served`
  - `dependency_missing_effort_edges`
  - `dependency_retry_pressure_missing`

# M-08.05 - MCP Edge Metrics Support

**Status:** Complete  
**Release Date:** 2026-01-21  
**Branch:** `milestone/m-08.05`

## Overview

M-08.05 aligns MCP with the edge-time-bin contract by exposing edge series, edge quality, and edge warnings directly from the engine. It also adds server-side edge filters on `/state_window` so MCP clients can request smaller, edge-focused payloads without deriving or mutating data client-side.

### Highlights
- **Edge-aware MCP tools:** `get_state_window` accepts edge filters and returns edge series plus edge warnings.
- **Edge quality surfaced:** MCP passes through `metadata.edgeQuality` without inference.
- **Doc + test coverage:** MCP docs and tests now reflect edge semantics and filtering.

## API Changes

- **`GET /v1/runs/{runId}/state_window`**
  - Added query params: `edgeIds`, `edgeMetrics`, `classIds`.
  - Edge warnings are filtered by `edgeIds` when provided.

## Tooling Updates

- MCP `get_state_window` schema includes `edgeIds`, `edgeMetrics`, and `classIds`.
- MCP responses include edge series and `edgeWarnings` when present.
- MCP docs updated to reference edge quality and edge series behavior.

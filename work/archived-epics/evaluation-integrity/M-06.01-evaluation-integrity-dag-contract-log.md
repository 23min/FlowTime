## Phase 1: Contract + Inventory

**Goal:** Document the evaluation contract and identify all post-eval mutation/override paths.

### Task 1.1: Inventory evaluation pipeline + override paths
**File(s):** TBD (identify engine eval + class propagation pipeline)

**Checklist (TDD Order - Tests FIRST):**
- [x] N/A (analysis-only)
- [x] Document evaluation pipeline and mutation points

**Findings (pipeline + override paths):**
- `src/FlowTime.Core/Routing/RouterAwareGraphEvaluator.cs` runs a two-pass evaluation: `Graph.Evaluate` then `Graph.EvaluateWithOverrides` using overrides from `RouterFlowMaterializer.ComputeOverrides` (full recompute path for totals).
- `src/FlowTime.Core/Routing/RouterFlowMaterializer.cs` builds router overrides and uses `ClassContributionBuilder.Build` to compute class series for class-aware routing.
- `src/FlowTime.Core/Artifacts/ClassContributionBuilder.cs` builds class series in DAG order, then applies router/serviceWithBuffer/topology overrides and **recomputes** contributions after each override batch (post-eval mutation + partial recompute risk).
- `src/FlowTime.Core/Artifacts/RunArtifactWriter.cs` mutates the evaluated totals context after evaluation:
  - `TryPrecomputeQueueDepthSeries` injects queue series from semantics + arrivals/served.
  - `TryPrecomputeRetryEchoSeries` injects retry echo series from failures/errors + retry kernel.
  - `ClassSeriesOverride` allows external class series injection **after** class contributions are built.
- `src/FlowTime.Core/Analysis/InvariantAnalyzer.cs` and `src/FlowTime.Sim.Core/Analysis/TemplateInvariantAnalyzer.cs` both call `ClassContributionBuilder.Build`, so any post-eval mutation in that pipeline affects diagnostics and warning fidelity.

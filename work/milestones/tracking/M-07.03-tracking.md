## Notes

### Key Decisions
- Edge quality now considers router class subsets as exact when per-edge class sets are satisfied.
- Edge lag is flagged as a warning to discourage edge-level behavior; transit should be modeled as nodes.
- Retry badge moved to edges and node badge disabled by default.

### Lessons Learned
- **What went well:** Edge overlay fidelity surfaced clearly with retry load and lag context.
- **What could be improved:** Full API test suite timeout needs a stable split or longer timeout.
- **Future considerations:** Replace lagged edges with explicit transit nodes.

### Dependencies Discovered
- Derived path latency needs explicit milestone (M-07.05).

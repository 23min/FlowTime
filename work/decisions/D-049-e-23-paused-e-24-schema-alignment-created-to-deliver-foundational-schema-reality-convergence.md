**Status:** active
**Context:** During M-046 Schema Alignment, the bisection-style investigation across the twelve templates under `templates/*.yaml` surfaced **16 distinct schema-vs-reality error shapes** (726 total validator errors), only one of which — `/grid/start: All values fail against the false schema` — the E-23 epic context had anticipated. The root causes cluster into five classifications, not a single "add missing field" pattern:

1. **Architectural seam (4 shapes, 48 errors):** `SimModelArtifact` is serialized directly as the `POST /v1/run` payload, mixing Sim-bundle traceability fields (`window`, `mode`, `metadata`, `generator`) with the Engine submission shape. The schema's root `additionalProperties: false` rejects them correctly; the defect is the conflation, not the schema.
2. **Systematic rule violation (4 shapes, 72 errors):** The `provenance` block in `docs/schemas/model.schema.yaml` declares snake_case field names (`generated_at`, `model_id`, `template_id`, `template_version`) while Sim's `SimProvenance` emits camelCase via YamlDotNet default conventions. Sim, the Engine reader, and the project rule ("JSON payloads and schemas use camelCase") all agree with camelCase; the schema is the inconsistent surface.
3. **Validator defect (4 shapes, ~231 errors):** `ModelSchemaValidator.ParseScalar` (`src/FlowTime.Core/Models/ModelSchemaValidator.cs:222-246`) and the mirrored `TemplateSchemaValidator.ParseScalar` (`src/FlowTime.Sim.Core/Templates/TemplateSchemaValidator.cs:173-197`) both coerce quoted numeric literals (e.g. `expr: "0"`) to JSON integers before schema evaluation, then fail the `type: string` declaration. YAML 1.2 node resolution distinguishes `ScalarStyle.Plain` (coercible) from quoted/block styles (must resolve as string); the validators ignore the distinction. Prior art for the fix exists at `src/FlowTime.Core/Models/ParallelismReference.cs:97`.
4. **Emitter drift (2 shapes, 375 errors):** Sim's `EnsureSemanticsOutputs` auto-adds `outputs[]` entries shaped `{id, series}` while the schema requires `{series, as}` with `as` as the required filename — this single method produces 366 of the 495 non-C-defect errors (74% of non-defect noise). Separately, Sim emits `classes: []` for templates with no class definitions while the schema declares `classes: minItems: 1` (9 errors).
5. **Emitter redundancy (1 shape):** `SimProvenance` emits `schemaVersion` inside the provenance block in addition to the top-level `schemaVersion`, and `provenance.mode` duplicates top-level `mode`.

Applying the user's explicit direction — **"absolutely correct, future-proof. No fixes because they are easy. Do the right thing."** — requires per-field design decisions on the correct resolution side (schema / Sim emitter / Engine parser / validator). This is substantially broader than E-23's consolidation thesis, which assumed the divergence was a single missing schema field and could be fixed by schema-widening alone. The survey agent that produced the evidence is `a07d52c12dcaf3538`; the full-shape audit is captured at `work/epics/E-23-model-validation-consolidation/m-E23-01-schema-alignment-tracking.md` → "AC4 canary re-run, full-shape audit (2026-04-24)".

**Decision:** Pause E-23 at M-046 and promote the foundational convergence work to a new epic **E-24 Schema Alignment** (`work/epics/E-24-schema-alignment/spec.md`) with six milestones: M-049 Inventory & Design Decisions (doc-only), M-050 Engine Submission Payload Separation (architectural), M-051 Schema Realignment (camelCase + decided shapes), M-052 Sim Emission Discipline (emitter corrections), M-053 Parser/Validator Scalar-Style Fix (mirrored in both validators), m-E24-06 Canary Green + Hard Assertion (close + E-23 resume gate). The uncommitted M-046 artefacts on branch `milestone/m-E23-01-schema-alignment` (three schema edits: `grid.start`, `nodes[].metadata`, `nodes[].source`; two `work/gaps.md` entries; the tracking doc) are stashed as preserved input material for M-049 — each schema edit is re-examined in M-049 under the foundational framework before any survives into M-051. E-23 resumes after E-24 closes; at that point M-047 (call-site migration) and M-048 (`ModelValidator` delete) become byte-trivial because `ModelSchemaValidator`, the schema, Sim's emitter, and the Engine's reader will actually agree. E-24 integration branch `epic/E-24-schema-alignment` created from `main`; milestone specs drafted; status surfaces reconciled.

**Consequences:** E-24 is the active epic. E-23 enters `paused` with `depends_on: E-24-schema-alignment`. M-046 spec remains `in-progress` in its frontmatter on the `milestone/m-E23-01-schema-alignment` branch (as stashed) — reconciliation of that file's status to `paused (pending E-24)` happens whenever the branch is next touched. Every foundational divergence identified by the survey gets a design decision recorded in M-049 before any code, schema, or emitter change lands. The "easy option" (schema pass-through widening) is explicitly rejected in favour of emitter correction or architectural separation on a per-field basis. The canary `TemplateWarningSurveyTests` promotes from informational diagnostic to hard-asserting regression guard at m-E24-06's close; a non-zero `val-err` thereafter is a failing build. Downstream: M-044 Validation Surface consumes the consolidated validator once both E-24 and E-23 close; E-15 Telemetry Ingestion starts from a ratified `nodes[].source` forward contract rather than inheriting a forward-declared placeholder. This decision directly applies the 2026-04-23 Truth Discipline guard at the schema layer: schema does not become a place where semantics live without Engine consumers; compatibility shims (snake_case aliases, `SimModelArtifact` on the wire, dual-shape acceptance) are rejected the same way they were rejected for validator delegation.

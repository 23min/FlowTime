**Status:** active
**Context:** Post-E-24 review of E-23's specification (2026-04-26) surfaced that the original M-046 milestone — titled "schema-alignment" — was largely absorbed by E-24 M-051 (`model.schema.yaml` rewrite) and M-053 (canary hard-assertion). What was *not* absorbed: the rule-by-rule audit of `ModelValidator.cs` that would prove `ModelSchemaValidator` + the schema together enforce every rule the deleted validator currently enforces. Without that audit, M-047 (call-site migration) and M-048 (`ModelValidator` delete) risk silently dropping rules that exist only as imperative checks in `ModelValidator.cs` (~25 rules across 214 lines). The user reframed the spirit of E-23 to its proper scope: *"After E24 and E23, hopefully our schema story is coherent and we don't have 'embedded schemas' anymore (different rules in different places)."* That reframing extends the audit scope beyond just `ModelValidator.cs` to every place where structural model rules live outside the canonical schema: parser tolerations in `ModelParser.cs`, silent emission defaults in `SimModelBuilder.cs` (post-E-24), and post-parse orchestration checks in `RunOrchestrationService.cs` and `GraphService.cs`. The original M-046 slug also collided with E-24's "schema-alignment" — a literal name collision that misframes both the work and the relationship between the epics.

**Decision:** Replace the original M-046 milestone with a new M-046 milestone. The new milestone is doc-first: enumerate every rule embedded outside the canonical schema (in `ModelValidator.cs`, `ModelParser.cs`, `SimModelBuilder.cs`, `RunOrchestrationService.cs`, `GraphService.cs`); for each rule record file:line, plain-English content, current schema status, and disposition; land schema additions (with citation comments) and `ModelSchemaValidator` adjuncts (alongside `ValidateClassReferences`) where the audit shows they are needed; commit a negative-case canary `RuleCoverageRegressionTests` that proves `ModelSchemaValidator` actually catches every audited rule before M-047 routes production traffic through it. No call-site changes in M-046. M-047 and M-048 stay structurally as drafted — M-047 refreshed to start with a fresh `grep` rather than relying on cached line numbers (E-24 rewrites shifted them); M-048 refreshed to reference both canaries (E-24's `TemplateWarningSurveyTests` survey canary + M-046's `RuleCoverageRegressionTests` negative-case canary) as the green-confirmation gate. Epic spec rewritten with explicit "Spirit and history" section reframing E-23's goal as eliminating *embedded schemas* generally, with `ModelValidator` deletion as the most visible instance. The pre-pivot stashed input material (branch `milestone/m-E23-01-schema-alignment` + `stash@{0}`) is now historical — most absorbed by E-24, none required as input for the new audit; should be retired when E-23 resumes. Branch deletion is held for human approval per project rule (destructive operation).

**Consequences:** E-23 retains three milestones but the slate is rebuilt around audit-first discipline, not schema-edit-first. The slug collision with E-24 is resolved (`m-E23-01-rule-coverage-audit` vs. E-24's "schema-alignment"). Future readers of E-23 see a clear spirit statement on the epic's first page that ties the work to the user's reframing: schema is the only declarative source, `ModelSchemaValidator` is the only runtime evaluator, every rule has exactly one canonical home. ADR-E-23-02 ("Schema as the single structural contract; adjuncts are the named exception") is now ratifiable at M-046 close with the enumerated adjunct list as the artefact; ADR-E-23-03 (parser/emitter rules need written justification) is ratified only if the audit produces a non-empty `parser-justified` set. Surface sync: epic spec rewritten; M-046 file replaced (new file at `m-E23-01-rule-coverage-audit.md`; old `m-E23-01-schema-alignment.md` deleted via `git rm`); M-047 and M-048 lightly refreshed; CLAUDE.md, ROADMAP.md, and epic-roadmap.md E-23 sections rewritten to reflect the rescope. The `milestone/m-E23-01-schema-alignment` git branch and `stash@{0}` are flagged for retirement at E-23 resume; the new milestone starts fresh from post-E-24 `main` with no stash dependency.

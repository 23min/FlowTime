# Skill: code-review

Purpose: review changes for correctness, regressions, and missing tests.

Use when:
- Asked for a review or before wrapping a milestone.

Checklist:
- Behavior: regressions, edge cases, error handling.
- Tests: coverage for new logic, deterministic behavior.
- Contracts: schema/API compatibility preserved.
- Docs: updated where behavior or contracts changed.
- Style: matches existing patterns; no broad refactors.
